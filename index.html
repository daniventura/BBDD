Fábrica E-Liquid

Planteamiento

Una fábrica de líquidos para vapear quiere guardar información e inventario de sus productos.

Los propios líquidos son el resultado de mezclar un aroma y una base, a esta composición se le puede añadir nicotina, la cual se registra según su concentración en mg por mL de líquido. Según la base y el aroma que mezclemos, el producto necesitará un determinado tiempo de maceración para conseguir un estado óptimo de consistencia y sabor. Los aromas pueden ser simples; registrados como aromas alimenticios, o pueden ser una mezcla de varios para conseguir sabores y matices distintos. A los aromas se les asigna un código breve que lo defina y se define dentro de una categoría entre frutal, postre, tabaquil o artificial.

Las bases son una mezcla de dos compuestos de los que se quiere saber la proporción entre ellos; glicerina vegetal para generar vapor y propilenglicol en calidad alimenticia para absorber el aroma. Además se le puede añadir ciertas “moléculas” que añadan sensaciones al vapear, como una sensación mentolada o un golpe de garganta para asemejarse a fumar tabaco. Un código identificará las distintas bases.

La fábrica maneja varios estándares de volúmenes en sus recipientes comerciales como son 5, 10, 50 y 100 mL, para todos los tamaños tienen una versión en plástico (chubby) y otra de cristal con una pipeta.

Al envasar el producto, ha de etiquetarse identificando al líquido por su nombre, como cada líquido puede ser comercializado en distintos recipientes, habrá etiquetas para cada líquido en los tamaños adecuados para el correspondiente envase. De esta etiqueta interesa saber los colores en CMYK que se necesitan para imprimirlas y la imagen de su diseño.

Diagrama Entidad-Relación

*Observaciones: los atributos de la interrelación "se mezcla", podrían ser perfectamente atributos de la entidad agregada “líquido”, pero con el fin de poder incluir en la tabla E-R ejemplos de entidad con atributos, he decidido representarlos de esta forma.

Correcciones y Ajustes Diagrama Entidad-Relación

*1: El nombre del líquido no bastaba para identificar al líquido, ya que un líquido puede tener el mismo nombre con concentraciones de nicotina diferentes. Por tanto, he implementado un atributo cod. de líquido.

*2: La cardinalidad era incorrecta en esta interrelación, ahora si se corresponde con nuestro caso, ya que un líquido puede formar parte de ninguno o varios aromas compuesto, de igual forma que un líquido puede formado por otros o por varios.

*3: Los atributos de estas entidades estaban puestos forzosamente, ya que en su momento creia que tenían que tener atributos obligatoriamente para que tuviese sentido la jerarquía, pero sabiendo que esto no tiene por que ser así, he decidido eliminarlos porque no tenian mucho sentido.

*4: De nuevo he añadido un atribunto código, en este caso para el recipiente, porque el volunen no termina de identificar esta entidad ya que puede haber recipientes de diferente material pero de igual volumen.

CREATE DATABASE fabricaELiquid;

Creamos las tablas

CREATE TABLE liquidoBasico (
nombreLiquido VARCHAR(20) PRIMARY KEY,
codBase VARCHAR(4) NOT NULL,
codAroma VARCHAR(4) NOT NULL
);

CREATE TABLE maceracion (
nombreLiquido VARCHAR(20),
concentracionNicotina ENUM('0,3mg/mL', '0,6mg/mL', '1.2mg/mL') NOT NULL,
tmacerado TINYINT NOT NULL,
CONSTRAINT Pkmaceracion PRIMARY KEY (nombreLiquido,concentracionNicotina),
CONSTRAINT Cktmaceracion CHECK (5<=tmacerado<=30)
);

CREATE TABLE liquidoFinal (
codLiquido VARCHAR(4) PRIMARY KEY,
nombreLiquido VARCHAR(20) NOT NULL,
concentracionNicotina ENUM('0,3mg/mL', '0,6mg/mL', '1.2mg/mL') NOT NULL,
CONSTRAINT UkliquidoFinal UNIQUE KEY (nombreLiquido,concentracionNicotina)
);

CREATE TABLE aroma (
codAroma VARCHAR(4) PRIMARY KEY,
aroma ENUM('frutal', 'postre', 'tabaquil', 'artificial') NOT NULL UNIQUE KEY,
categoria VARCHAR(10)
);

CREATE TABLE base (
codBase VARCHAR(4) PRIMARY KEY,
proporcion VARCHAR(6) NOT NULL,
CONSTRAINT Ckproporcion CHECK (proporcion REGEXP '(100|[0-9]{2})%VG')
);

CREATE TABLE moleculas (
molecula ENUM('frescor', 'golpe de garganta'),
codBase VARCHAR(4),
CONSTRAINT Pkmoleculas PRIMARY KEY (molecula,codBase)
);

CREATE TABLE mezclaAromas (
aroma VARCHAR(4),
aromaCompuesto VARCHAR(4),
CONSTRAINT PkmezclaAromas PRIMARY KEY (aroma,aromaCompuesto)
);

CREATE TABLE recipiente (
codRecipiente VARCHAR(4) PRIMARY KEY,
volumen ENUM('5mL', '10mL', '50mL', '100mL') NOT NULL,
tipo ENUM('chubby', 'cristal') NOT NULL DEFAULT 'chubby',
CONSTRAINT Ukrecipiente UNIQUE KEY (volumen,tipo)
);

CREATE TABLE envasado (
codLiquido VARCHAR(4),
codRecipiente VARCHAR(4),
diseño BINARY NOT NULL,
CONSTRAINT Pkenvasado PRIMARY KEY (codLiquido,codRecipiente)
);

CREATE TABLE colores (
color VARCHAR(15),
codLiquido VARCHAR(4),
codRecipiente VARCHAR(4),
CONSTRAINT Pkcolores PRIMARY KEY (color,codLiquido,codRecipiente),
CONSTRAINT Ckcolor CHECK (color REGEXP '((100)|([0-9]{2}),)((100)|([0-9]{2}),)((100)|([0-9]{2}),)((100)|([0-9]{2}))')
);
